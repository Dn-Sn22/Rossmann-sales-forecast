# Прогнозирование продаж сети Rossmann

Прогноз дневной выручки магазинов дрогери-сети **Rossmann** (Kaggle, 1115 магазинов
в Германии, 2013-2015). Лучшая модель развёрнута двумя способами: REST-сервис на
**FastAPI** и интерактивное веб-демо на **Gradio** (Hugging Face Spaces).

Проект проходит полный цикл DS: EDA -> feature engineering -> сравнение моделей
(XGBoost / LSTM / GRU) -> эксперименты и тюнинг -> продакшн-инференс.

**Живое демо:** https://huggingface.co/spaces/Dalen22/rossmann-sales-forecast

---

## Результаты

Сравнение на временной валидации (обучение на данных до 2015-06-19, проверка - после):

| Модель                | MAE    | RMSE    | MAPE   |
|-----------------------|--------|---------|--------|
| XGBoost (baseline)    | 662.93 | 950.34  | 10.18% |
| LSTM                  | 756.49 | 1057.59 | 10.12% |
| GRU                   | 923.97 | 1271.65 | 12.45% |
| **XGBoost (tuned)** ⭐ | **616.55** | **883.16** | **9.42%** |

**Финальная модель - настроенный XGBoost** (MAPE 9.42%). Для табличных данных
с качественным feature engineering градиентный бустинг превзошёл рекуррентные сети.

## Ключевые решения

- **Временной holdout** (split по дате, не случайный) - имитирует реальную задачу
  «прогноз будущего по прошлому» и исключает утечку из будущего.
- **Признаки-лаги и скользящие средние** (7/14/28 дней) со сдвигом `shift(1)` -
  дают модели историю продаж без подглядывания в текущий день.
- **Рекурсивный прогноз на test**: горизонт ~6 недель, а лаги короче, поэтому
  предсказание идёт по дням с подстановкой прогноза обратно в историю.
- **эксперименты:** логарифмирование таргета (`log1p`) и адаптивный
  learning rate (`ReduceLROnPlateau`) проверены и отклонены поскольку стабильного
  улучшения не дали, финалом остался baseline + тюнинг XGBoost.

## Пайплайн (ноутбук)

1. **EDA** - распределение продаж (right-skewed), сезонность, влияние промо и типа магазина, корреляции.
2. **Feature Engineering** - обработка пропусков, признаки из даты, `IsPromoMonth`, кодирование категорий, лаги и скользящие средние. Итог: ~813 000 строк, 24 признака.
3. **Baseline** - XGBoost на дефолтных параметрах как точка отсчёта.
4. **LSTM / GRU** - последовательности по 30 дней, масштабирование, сравнение с бустингом.
5. **Эксперименты** - `log1p`, `ReduceLROnPlateau`.
6. **Тюнинг XGBoost** - случайный поиск гиперпараметров на временном holdout.
7. **Прогноз на test + сабмит** - рекурсивная генерация `submission.csv` для Kaggle.

## Запуск

Установка зависимостей:

```bash
pip install -r requirements.txt
```

### Веб-демо (Gradio)

```bash
python app.py
```

Открывается на `http://127.0.0.1:7860`: выбираете магазин, дату и флаги - сервис сам
считает признаки из истории продаж и показывает прогноз (а для известных дат - ещё
и реальное значение с процентом ошибки).

### REST API (FastAPI)

```bash
uvicorn main:app --reload
```

Интерактивная документация: **http://127.0.0.1:8000/docs**

### Эндпоинты

| Метод | Путь       | Описание                         |
|-------|------------|----------------------------------|
| GET   | `/health`  | Проверка работоспособности       |
| POST  | `/predict` | Прогноз продаж по признакам дня  |

### Пример запроса

```bash
curl -X POST http://127.0.0.1:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "Store": 1, "DayOfWeek": 5, "Open": 1, "Promo": 1,
    "StateHoliday": "0", "SchoolHoliday": 1,
    "StoreType": "c", "Assortment": "a",
    "CompetitionDistance": 1270,
    "CompetitionOpenSinceMonth": 9, "CompetitionOpenSinceYear": 2008,
    "Promo2": 0, "Promo2SinceWeek": 0, "Promo2SinceYear": 0,
    "Year": 2015, "Month": 7, "Day": 31, "WeekOfYear": 31,
    "IsPromoMonth": 0,
    "lag_7": 5263, "lag_14": 5020, "lag_28": 4890,
    "rolling_mean_7": 5100, "rolling_mean_28": 5000
  }'
```

Ответ:

```json
{ "predicted_sales": 5929.87 }
```
